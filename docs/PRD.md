## 二、功能需求详细说明

### 2.1 新增工具:密码生成器

#### 2.1.1 功能概述
提供可自定义的随机密码生成服务,支持多种字符组合规则,帮助用户创建高强度密码。

#### 2.1.2 功能需求

**核心功能**

| 功能项 | 需求描述 | 优先级 | 验收标准 |
|--------|---------|--------|---------|
| 密码长度设置 | 支持8-32位可调节,默认16位 | P0 | 滑动条/输入框可调节,实时显示当前值 |
| 大写字母开关 | 包含A-Z字符 | P0 | 开关状态可切换,默认开启 |
| 小写字母开关 | 包含a-z字符 | P0 | 开关状态可切换,默认开启 |
| 数字开关 | 包含0-9字符 | P0 | 开关状态可切换,默认开启 |
| 特殊符号开关 | 包含!@#$%^&*等符号 | P0 | 开关状态可切换,默认开启 |
| 密码生成 | 根据设置生成随机密码 | P0 | 点击按钮立即生成,显示在结果区 |
| 一键复制 | 复制生成的密码到剪贴板 | P0 | 点击按钮复制成功,显示提示信息 |
| 密码强度指示 | 实时显示密码强度等级 | P1 | 弱/中/强三级显示,颜色区分 |
| 历史记录 | 保存最近10条生成记录 | P2 | 可查看历史,支持复制,可清空 |

**输入输出逻辑**

```
输入参数:
- length: number (8-32)
- uppercase: boolean
- lowercase: boolean
- numbers: boolean
- symbols: boolean

输出结果:
- password: string (生成的密码)
- strength: 'weak' | 'medium' | 'strong'
- timestamp: number (生成时间戳)
```

**密码生成规则**

1. **字符池构建**
   - 大写字母池: `ABCDEFGHIJKLMNOPQRSTUVWXYZ`
   - 小写字母池: `abcdefghijklmnopqrstuvwxyz`
   - 数字池: `0123456789`
   - 特殊符号池: `!@#$%^&*()_+-=[]{}|;:,.<>?`

2. **生成算法**
   ```
   步骤1: 根据开关状态构建可用字符池
   步骤2: 确保至少包含每个开启类型各1个字符
   步骤3: 从字符池随机选取剩余字符
   步骤4: 打乱字符顺序
   步骤5: 返回最终密码
   ```

3. **强度计算规则**
   - **弱**: 长度<12 或 仅包含1种字符类型
   - **中**: 长度12-16 且 包含2-3种字符类型
   - **强**: 长度>16 且 包含4种字符类型

**用户交互流程**

```
开始
  ↓
[用户进入密码生成器页面]
  ↓
[系统加载默认配置:16位,全类型开启]
  ↓
[用户调整参数] ←—————┐
  ↓                  │
[点击"生成密码"按钮]  │
  ↓                  │
[系统生成密码]        │
  ↓                  │
[显示密码+强度指示]   │
  ↓                  │
[用户满意?] —否—→———┘
  │是
  ↓
[点击"复制"按钮]
  ↓
[系统复制到剪贴板]
  ↓
[显示"复制成功"提示]
  ↓
[保存到历史记录]
  ↓
结束
```

**异常处理**

| 异常场景 | 处理方式 |
|---------|---------|
| 未选择任何字符类型 | 提示"请至少选择一种字符类型",禁止生成 |
| 长度输入超出范围 | 自动修正到边界值(8或32) |
| 复制失败(浏览器不支持) | 提示"当前浏览器不支持复制,请手动选择复制" |
| 历史记录存储失败 | 提示"历史记录保存失败",不影响主功能 |

## 三、数据规范定义

### 3.1 localStorage 存储键名规范

**命名规则**
```
格式: {project}_{module}_{subkey}
示例: toolbox_calculator_history
```

**统一键名清单**

| 键名 | 用途 | 数据类型 | 说明 |
|------|------|---------|------|
| `toolbox_global_theme` | 全局主题设置 | string | 'light' \| 'dark' |
| `toolbox_global_stats` | 全局统计数据 | object | 各工具使用统计 |
| `toolbox_calculator_history` | 计算器历史记录 | array | 最近20条计算记录 |
| `toolbox_todo_tasks` | 待办任务列表 | array | 所有待办任务 |
| `toolbox_password_config` | 密码生成器配置 | object | 用户偏好设置 |
| `toolbox_password_history` | 密码生成历史 | array | 最近10条记录 |
| `toolbox_recent_used` | 最近使用记录 | array | 最近使用的工具列表 |

### 3.2 各工具数据持久化规则

#### 3.2.1 计算器数据

**数据结构**
```json
{
  "history": [
    {
      "id": "string",
      "expression": "string",
      "result": "number",
      "timestamp": "number"
    }
  ],
  "maxCount": 20
}
```

**持久化规则**
- 每次计算完成后自动保存
- 最多保留20条记录,超出时删除最旧记录
- 支持手动清空历史

#### 3.2.2 待办清单数据

**数据结构**
```json
{
  "tasks": [
    {
      "id": "string",
      "content": "string",
      "completed": "boolean",
      "createdAt": "number",
      "completedAt": "number | null",
      "priority": "high" | "medium" | "low"
    }
  ],
  "version": "1.0"
}
```

**持久化规则**
- 添加/删除/修改/完成任务后立即保存
- 无数量限制(建议≤100条)
- 支持按完成状态筛选

#### 3.2.3 密码生成器数据

**配置数据结构**
```json
{
  "config": {
    "length": 16,
    "uppercase": true,
    "lowercase": true,
    "numbers": true,
    "symbols": true
  }
}
```

**历史记录数据结构**
```json
{
  "history": [
    {
      "id": "string",
      "password": "string",
      "config": {
        "length": "number",
        "types": "string[]"
      },
      "strength": "string",
      "timestamp": "number"
    }
  ],
  "maxCount": 10
}
```

**持久化规则**
- 配置变更后立即保存
- 生成密码后保存到历史
- 历史记录最多保留10条
- 注意:历史记录中保存明文密码,需在UI中提示用户安全风险

### 3.3 JSON导入导出全量备份/恢复数据格式标准

**导出文件命名**
```
toolbox_backup_YYYYMMDD_HHmmss.json
示例: toolbox_backup_20260603_143025.json
```

**完整数据格式**
```json
{
  "version": "3.0",
  "exportTime": "2026-06-03T14:30:25.000Z",
  "appInfo": {
    "name": "多功能工具箱",
    "version": "3.0.0"
  },
  "data": {
    "global": {
      "theme": "light",
      "stats": {
        "calculator": {
          "useCount": 50,
          "lastUsed": "2026-06-03T14:00:00.000Z"
        },
        "todo": {
          "useCount": 30,
          "lastUsed": "2026-06-03T13:00:00.000Z",
          "completedRate": 0.75
        },
        "password": {
          "useCount": 20,
          "lastUsed": "2026-06-03T12:00:00.000Z"
        }
      },
      "recentUsed": ["calculator", "todo", "password"]
    },
    "calculator": {
      "history": [
        {
          "id": "calc_001",
          "expression": "1+2+3",
          "result": 6,
          "timestamp": 1717422000000
        }
      ]
    },
    "todo": {
      "tasks": [
        {
          "id": "task_001",
          "content": "完成PRD文档",
          "completed": true,
          "createdAt": 1717422000000,
          "completedAt": 1717425600000,
          "priority": "high"
        }
      ]
    },
    "password": {
      "config": {
        "length": 16,
        "uppercase": true,
        "lowercase": true,
        "numbers": true,
        "symbols": true
      },
      "history": [
        {
          "id": "pwd_001",
          "password": "Abc123!@#xyz",
          "config": {
            "length": 12,
            "types": ["uppercase", "lowercase", "numbers", "symbols"]

          "strength": "medium",
```

**导入验证规则**
1. 检查version字段是否存在
2. 检查data结构是否完整
3. 版本兼容性检查(向下兼容)
4. 数据格式校验(字段类型、必填项)
5. 异常数据跳过并记录日志

**导入策略**
- 提供两种模式:
  - **覆盖模式**: 清空现有数据,导入备份数据
  - **合并模式**: 保留现有数据,追加备份数据(去重)

## 四、体验需求落地

### 4.1 数据统计面板设计

#### 4.1.1 面板位置
- 在导航栏右侧添加统计图标
- 点击展开统计面板(抽屉式/模态框)

#### 4.1.2 统计指标

| 指标名称 | 计算方式 | 展示形式 |
|---------|---------|---------|
| 计算器使用次数 | 累计计算次数 | 数字+趋势图标 |
| 待办任务总数 | 当前任务数量 | 数字 |
| 待办完成率 | 已完成/总数×100% | 进度条+百分比 |
| 密码生成次数 | 累计生成次数 | 数字 |
| 最常用工具 | 使用次数最多的工具 | 工具名称+图标 |

#### 4.1.3 数据更新时机
- 计算器:每次计算完成+1
- 待办:添加/完成任务时更新
- 密码:每次生成+1
- 页面加载时读取localStorage

### 4.2 深色/浅色主题切换方案

#### 4.2.1 切换入口
- 导航栏右侧主题切换按钮(太阳/月亮图标)
- 支持快捷键:Ctrl/Cmd + Shift + T

#### 4.2.2 主题定义

**浅色主题**
```css
--bg-primary: #ffffff
--bg-secondary: #f5f5f5
--text-primary: #333333
--text-secondary: #666666
--border-color: #e0e0e0
--accent-color: #1890ff
```

**深色主题**
```css
--bg-primary: #1a1a1a
--bg-secondary: #2d2d2d
--text-primary: #e0e0e0
--text-secondary: #a0a0a0
--border-color: #404040
--accent-color: #40a9ff
```

#### 4.2.3 切换逻辑
```
1. 用户点击切换按钮
2. 切换主题状态(light ↔ dark)
3. 保存到localStorage
4. 更新CSS变量
5. 添加过渡动画(300ms)
```

#### 4.2.4 初始化逻辑
```
1. 读取localStorage中的主题设置
2. 如无设置,检测系统主题偏好(prefers-color-scheme)
3. 应用对应主题
```

### 4.3 常用功能快捷入口设计

#### 4.3.1 入口位置
- 首页顶部快捷入口区
- 或导航栏下拉菜单

#### 4.3.2 快捷功能列表

| 功能名称 | 图标 | 操作 |
|---------|------|------|
| 快速计算 | 🔢 | 跳转到计算器 |
| 新建待办 | ✅ | 跳转到待办清单 |
| 生成密码 | 🔐 | 跳转到密码生成器 |
| 导出数据 | 💾 | 触发全量导出 |
| 切换主题 | 🌓 | 切换深浅主题 |

#### 4.3.3 显示逻辑
- 根据用户使用频率动态排序
- 最多显示5个快捷入口
- 移动端显示为横向滚动列表

### 4.4 最近使用记录的产品逻辑

#### 4.4.1 记录规则
- 记录用户最近访问的工具(最多5个)
- 每次切换Tab时更新记录
- 记录包含:工具名称、访问时间

#### 4.4.2 展示位置
- 首页"最近使用"模块
- 或导航栏"最近"下拉菜单

#### 4.4.3 数据结构
```json
{
  "recentUsed": [
    {
      "tool": "calculator",
      "lastUsed": "2026-06-03T14:00:00.000Z"
    },
    {
      "tool": "todo",
      "lastUsed": "2026-06-03T13:00:00.000Z"
    }
  ]
}
```

#### 4.4.4 交互逻辑
- 点击最近使用项 → 跳转到对应工具
- 悬停显示最后使用时间
- 支持清空最近记录

## 五、视觉规范统一

### 5.1 设计语言定义

#### 5.1.1 设计原则
- **简洁**: 减少视觉噪音,突出核心功能
- **一致**: 统一交互模式,降低学习成本
- **友好**: 清晰的反馈,舒适的色彩

#### 5.1.2 色彩规范

**主色调**
- 主色: #1890ff (蓝色)
- 成功色: #52c41a (绿色)
- 警告色: #faad14 (橙色)
- 错误色: #f5222d (红色)

**中性色**
- 标题: #262626
- 正文: #595959
- 辅助文字: #8c8c8c
- 边框: #d9d9d9
- 分割线: #f0f0f0

#### 5.1.3 字体规范

**字体家族**
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC',
  'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica,
  Arial, sans-serif;
```

**字号规范**
- 大标题: 24px / 1.4
- 小标题: 18px / 1.5
- 正文: 14px / 1.6
- 辅助文字: 12px / 1.5

#### 5.1.4 间距规范
- 超小间距: 4px
- 小间距: 8px
- 中间距: 16px
- 大间距: 24px
- 超大间距: 32px

#### 5.1.5 圆角规范
- 小圆角: 4px (按钮、标签)
- 中圆角: 8px (卡片、输入框)

- 高度: 40px
- 内边距: 8px 12px
- 圆角: 8px

#### 5.2.3 卡片组件
- 内边距: 16px
- 圆角: 8px
- 阴影: 0 2px 8px rgba(0,0,0,0.1)

### 5.3 移动端适配规范

#### 5.3.1 响应式断点
```css
/* 移动端 */
```

## 七、技术实现建议

### 7.1 技术栈建议

**前端框架**
- 原生HTML/CSS/JavaScript(轻量级)
- 或 Vue 3 / React(如需复杂交互)

**样式方案**
- CSS变量实现主题切换
- Flexbox/Grid布局
- 媒体查询实现响应式

**数据存储**
- localStorage(客户端持久化)
- IndexedDB(如需存储大量数据)

**工具库**
- crypto.getRandomValues()(密码生成)
- Clipboard API(复制功能)

### 7.2 性能优化建议

- 防抖/节流处理频繁操作
- 虚拟滚动处理长列表
- 懒加载非关键资源
- 压缩静态资源
- 使用CDN加速

### 7.3 安全性建议

- 密码生成使用加密随机数
- localStorage数据不存储敏感信息
- XSS防护:转义用户输入
- 外部链接使用rel="noopener noreferrer"
